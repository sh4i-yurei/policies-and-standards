# Engineering Knowledge Base

Single source of truth for engineering standards, workflows, design
frameworks, and project templates.

## Getting started

Start with [Onboarding](00_Orientation/Onboarding.md) for the recommended
reading path and KB entry point.

## Directory structure

| Directory | Contents |
|-----------|----------|
| `00_Orientation/` | Mission, principles, vision, terminology, onboarding |
| `01_Governance/` | Governance overview, standards policy, issue management, release policy |
| `02_Documentation_Standards/` | Documentation standard (STD-001), vault schema |
| `03_Engineering_Standards/` | Coding, testing, security, AI, tooling, SLI/SLO, and AI rule packs |
| `04_Design_Framework/` | Design-first model, system/module/spec standards, review checklist |
| `05_Dev_Workflows/` | SDLC, git workflow, CI/CD, design review, release, incident response |
| `06_Projects/Templates/` | All project templates (planning, design, architecture, AI, ops, release) |

## Contributing

Follow the [documentation change workflow](05_Dev_Workflows/documentation_change_workflow.md).
All changes go through pull requests per the
[git and branching workflow](05_Dev_Workflows/git_and_branching_workflow.md).

## Document structure

Every governed document follows the 7-section structure defined in the
[Documentation Standard](02_Documentation_Standards/Documentation_Standard.md):
Purpose, Scope, Standard, Implementation Notes, Continuous Improvement,
Compliance, Changelog.
