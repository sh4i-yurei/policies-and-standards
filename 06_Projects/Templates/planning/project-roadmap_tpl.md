# Purpose

Provide a project-level roadmap that summarizes scope, sequencing, and
milestones for multi-step work.

# Scope

Use for Tier 2 and Tier 3 initiatives or any project with more than one
milestone.

# Standard

## Roadmap snapshot

- Project name: <name>
- Owner: <name/contact>
- Change tier: <1|2|3>
- Current phase:
  <initiation|design|specification|implementation|validation|release|operate>
- Target horizon: <quarter or date range>

## Goals and outcomes

List measurable outcomes and success criteria.

## Milestones and releases

- Milestone: <name>
  - Target date: <date>
  - Success criteria: <text>
  - Release target: <version or tag>
  - Dependencies: <links>

## Work streams

- Work stream: <name>
  - Scope: <text>
  - Owner: <name>
  - Linked backlog items: <links>

## Risks and constraints

List top risks, constraints, and mitigation owners.

## Dependencies

List external dependencies and required approvals.

## Status updates

- Date: <date>
  - Progress summary: <text>
  - Changes to scope: <text>
  - Next actions: <text>

## Links

- Intake or issue: <link>
- PRD: <link>
- Charter: <link>
- Backlog/milestones: [backlog-milestones_tpl](backlog-milestones_tpl.md)
- Specs and designs: <links>
- Release plan: <link>

# Implementation Notes

- Keep this roadmap lightweight and update it at major milestones.
- Use the backlog template for detailed task tracking.

# Continuous Improvement and Compliance Metrics

- Track roadmap accuracy versus delivered milestones.

# Compliance

Tier 2 and Tier 3 projects without a roadmap are non-compliant.
